# 고추밭 내전 경매 시스템 v4.0.4

회원가입이나 관전자 페이지 없이, 필요할 때 일회성 경매 작업방을 만들어 사용하는 버전입니다.

## 사용 흐름

1. 사이트에서 **작업방 시작**을 누릅니다.
2. 영문/숫자 방 코드와 비밀번호를 입력해 새 작업방을 만듭니다.
3. 선수, 팀, 설정, 룰렛, 낙찰·유찰, 로그를 해당 방에서 관리합니다.
4. 브라우저나 PC를 꺼도 같은 방 코드와 비밀번호로 다시 접속할 수 있습니다.
5. 방 데이터는 다른 방과 완전히 분리됩니다.
6. 방 삭제 시 비밀번호를 다시 입력해야 합니다.
7. 마지막 사용 후 30일이 지난 방은 방 목록을 불러올 때 자동 정리됩니다.

## 이번 버전 변경점

- 기본 제목을 `고추밭 내전`으로 변경
- 상단/경매 화면 문구를 `고추밭 내전 경매 시스템`으로 변경
- 기본 샘플 선수 데이터 제거
- 새 작업방은 선수 0명, 로그 0개인 빈 상태로 시작
- 비밀번호 관련 문구를 간단하게 정리
- 삭제 시 방 코드 입력 대신 비밀번호 재확인
- 로컬 화면이 아닌 Supabase 임시 작업방 중심으로 사용
- 30일 미사용 작업방 자동 정리 기능 추가

## 기존 배포에 적용

1. Supabase SQL Editor에서 `supabase/v404-temporary-room-cleanup.sql`을 한 번 실행합니다.
2. 이 프로젝트를 GitHub 저장소에 덮어씌우고 커밋/푸시합니다.
3. Vercel에서 새 배포가 완료되면 사용합니다.

환경 변수는 기존과 동일합니다.

- `NEXT_PUBLIC_SUPABASE_URL`
- `NEXT_PUBLIC_SUPABASE_ANON_KEY`
